# Jouleverse节点包RC20260129测试发布

**致 Jouleverse 社区全体节点运维者：**

Jouleverse 主网节点软件包 **RC20260129** 版本现已发布并进入社区测试阶段。此版本是对 2024 版节点架构的一次重要升级，旨在提升节点运维的便捷性、安全性与稳定性。

**建议所有记账节点（Miner Node）尽快安排升级测试**，以确保网络的平稳过渡和安全性。见证节点（Witness Node）也建议同步升级，以获得最佳体验。

## **RC20260129 版本核心改进**

相较于 RC20240229 版本，新版本节点包引入了以下关键改进：

1. **集成 Clef 签名机，增强安全性**
    * **革新的安全架构：** 记账节点不再将 keystore 和明文密码文件直接存放在数据目录中。新版本集成了 Clef 签名机，对签名密钥进行加密托管，实现了签名操作与节点运行进程的分离，大幅降低了签名私钥泄露风险。
    * **引导式初始化：** 通过全新的管理脚本，首次搭建记账节点将进入一个交互式、步骤清晰的 Clef 初始化流程，指导用户安全地设置加密密码和账户规则，降低了操作失误的可能性。

2. **全新一体化管理脚本，简化运维**
    * **封装管理脚本：** 无论是见证节点还是记账节点，现在都拥有专属的一键管理脚本（`jnode_witness` / `jnode_miner`）。所有日常操作，如启动 (`jnode start`)、停止 (`jnode stop`)、查看状态 (`jnode status`)、查看日志 (`jnode watch`)、进入控制台 (`jnode console`) 等，均通过简单的 `jnode` 命令完成，告别冗长的 Docker 命令。
    * **标准化流程：** 脚本将初始化、备份、升级、清理等流程标准化，减少了因手动输入命令错误导致的问题（如误初始化数据目录），使节点管理变得更加简单和可靠。

3. **优化审计与连接体验**
    * 管理脚本内置了查看区块高度、节点信息等常用审计命令，方便快速提供审计所需信息。
    * 更新了审计节点的连接地址，确保新节点能够快速、稳定地连接到核心网络。

为确保主网顺利升级，我们呼吁社区节点运营者，特别是**记账节点运营者**，尽快按照 **[《Jouleverse Mainnet 节点搭建指南（2026版）》](https://wiki.jouleverse.com/#!/docs/network/how-to-setup-jouleverse-node.md)** 进行升级测试。

**重点提示：**

* **记账节点升级须知：** 从 2024 版升级至 2026 版，由于架构变更，**需要执行一次 Clef 签名机的初始化流程**，但会保留原有的区块链数据。请务必仔细阅读指南中“升级记账节点”部分的特别说明，并建议在操作前备份整个 `~/data` 目录。
* **版本验证：** 升级后，请使用 `jnode console admin.nodeInfo | grep name` 命令验证节点版本，正确输出应包含 **`tag-m20260129`**（记账节点）或 **`tag-w20260129`**（见证节点）。
* **测试反馈：** 在测试过程中遇到任何问题，或有任何改进建议，请在社群中反馈或提交github issuea到[节点包仓库](https://github.com/Jouleverse/jouleverse-node/issues)。
